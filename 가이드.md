## 2. Stage: econ – 창원 상장사 선택 + 지표/스토리 설계

### 2.1 대상 기업 후보 & 선정

**창원 본사 상장사 예시 후보**

* **두산에너빌리티(034020, 구 두산중공업)**

  * 본사: 경남 창원시 성산구 두산볼보로 22([Company Guide][1])
  * 업종: 발전 설비·가스터빈·원자력·플랜트(기계·장비)([나이스비즈정보][2])
  * 공시/재무: DART, FnGuide, WISEReport 등에서 연결/별도 재무제표, 재무비율 풍부하게 제공([Company Guide][1])

* **HD현대마린엔진(구 HSD엔진, 향후 한화엔진)**

  * 창원 소재 선박엔진 전문 제조사, 한화 그룹 편입·사명 변경 진행([한국경제][3])

* 기타: 한국철강, 무학, S&T중공업 등 (경남·창원 상장사 리스트 참조)([창원상공회의소][4])

**교육/면접 포트폴리오 관점 선택**

* **주 대상 기업**: **두산에너빌리티(Doosan Enerbility)**

  * 이유:

    * 글로벌 인프라·발전·원자력 등 “정책/금리/환율과 연결되는 산업 구조” → 거시와 여신을 묶기 좋음
    * 연결/별도 재무, 재무비율, 사업보고서가 잘 정리되어 있어 데이터 소스 설계가 용이([두산 에너빌리티][5])
    * 창원에 본사를 둔 대표 대기업이라 “지역+산업+공공성” 설명이 자연스러움

원하면 나중에 **보조 사례**로 HD현대마린엔진(구 HSD엔진) 한 줄 비교를 붙이는 구조로 확장 가능.

### 2.2 필수 재무 지표 세트 (기업여신 관점)

**(1) 손익 중심 지표**

* 매출액(Revenue) – `sales`
* 매출총이익(Gross profit) – `gross_profit`
* 영업이익(Operating income) – `op_income`
* EBITDA – `ebitda`
* 당기순이익(Net income) – `net_income`
* 영업이익률, EBITDA 마진, 순이익률 (%, `op_margin`, `ebitda_margin`, `net_margin`)

**(2) 재무상태(안정성) 지표**

* 총자산, 총부채, 자본 – `total_assets`, `total_liab`, `equity`
* 부채비율 = 부채/자본
* 유동자산, 유동부채 → 유동비율
* 비유동자산, 비유동부채 → 고정비율, 장기부채비율

**(3) 이자상환·현금흐름 중심 지표**

* 이자비용(Interest expense) – `interest_expense`
* 이자보상배율 = (영업이익 또는 EBITDA) / 이자비용
* 영업활동현금흐름(OCF), 투자활동CF, 재무활동CF – `cf_op`, `cf_inv`, `cf_fin`
* CAPEX (유형자산 취득 등) – `capex`
* 잉여현금흐름(FCF) = OCF – CAPEX

**(4) 성장성 지표**

* 매출 성장률, 영업이익 성장률, 순이익 성장률
* 자산·자본 성장률

### 2.3 여신 심사 스토리라인 축(Template)

**강점 축 예시**

* 글로벌 발전·원자력·플랜트 수주 기반의 **장기 캐시플로우 확보 가능성**
* 창원 지역에 대규모 제조·기술 인력을 보유한 **기반 산업 앵커 기업** 역할
* 기술 경쟁력(초초임계압 발전, 가스터빈, 신재생·수소 등)과 정부 정책(에너지·원전 정책) 연계([나이스비즈정보][2])

**리스크 축 예시**

* 수주산업 특성상 **프로젝트 단위 변동성** (대형 수주/손실, 공사손실/충당부채 등)
* 높은 설비·CAPEX 요구 → 경기 둔화 시 레버리지·현금흐름 부담
* 원자력·석탄 등 **정책·규제·ESG 이슈**에 따른 중장기 리스크

이 스토리 축은 나중에:

* `analytics.py` 의 **“여신 평가 요약”**
* `report_md.py` 의 **장점/리스크 섹션**
* `interview` 스테이지의 **스피치 스크립트**

에서 반복 재사용.

## 3. Stage: data – 공공 데이터 소스·수집 전략

### 3.1 Data Source 맵

| 구분   | 기관/플랫폼                               | 활용 예                  | 포맷/특징                |
| ---- | ------------------------------------ | --------------------- | -------------------- |
| 공시기본 | 금융감독원 DART                           | 사업보고서, 재무제표(연·분기), 주석 | HTML, XBRL, PDF, XLS |
| 재무요약 | FnGuide, WISEReport, Fnguide API(유료) | 요약 재무제표, 재무비율, 추세     | 웹/CSV/API            |
| 거래소  | KRX KIND                             | 상장사 기본 정보, 공시 링크      | 웹/CSV                |
| 기타   | NICE Bizinfo 등                       | 산업분류, 동종업계 비교 등       | 웹                    |

(실전 코드에서는 **DART + 수기/반자동 CSV** 조합으로 시작하는 게 현실적)

### 3.2 수집 단위 설계

* **기간**: 최근 **5~10개 결산연도** + 필요 시 분기(연결/별도 중 택1, 기본은 연결 IFRS)
* **종류**:

  * 재무상태표(Balance sheet)
  * 포괄손익계산서(Income statement)
  * 현금흐름표(Cashflow statement)
* **형태**:

  * 초기 버전: 직접 다운받은 엑셀/CSV에서 필요한 항목만 추출 → `data_raw/` 저장
  * 이후: DART Open API 연동(프로젝트 확장 옵션)

### 3.3 입력 파일/테이블 구조 (Template)

* `data_raw/034020_fs_income_2018_2024.csv`
* `data_raw/034020_fs_balance_2018_2024.csv`
* `data_raw/034020_fs_cashflow_2018_2024.csv`

각 파일은 최소 필드:

```text
fiscal_year, fiscal_type, account_code, account_name_kr, account_name_en, value_mil_krw
```

→ `etl.py` 에서 피벗/매핑하여 schema 단계 테이블로 변환.

### 3.4 데이터 품질/결측 처리 체크리스트

* 단위: 억원 vs 백만원 vs 천원 → `unit` 필드 표준화
* IFRS 특이 계정(기타포괄손익, 지배/비지배 지분 등) → 핵심 지표 계산에 어떻게 반영할지 정책 정의
* 일회성 손익(대규모 평가손실, 자산손상 등) →

  * `non_recurring_flag` 컬럼으로 표시
  * “보수 시나리오”에서 제거/감안하는 방식

## 4. Stage: schema – 재무 DB 설계 (Data Contract)

### 4.1 논리 모델

1. **companies**

* `company_id` (PK, 예: "034020")
* `name_kr`, `name_en`
* `hq_city`, `hq_region` (예: 창원, 경남)
* `sector_ksic`, `sector_name`
* `listing_market` (KOSPI/KOSDAQ)
* `fiscal_year_end` (예: 12)

2. **fs_income**

* `company_id`, `fiscal_year`, `fiscal_type`(A/Q)
* `sales`, `gross_profit`, `op_income`, `ebitda`, `net_income`, `interest_expense`, …

3. **fs_balance**

* `company_id`, `fiscal_year`, `fiscal_type`
* `total_assets`, `total_liab`, `equity`, `current_assets`, `current_liab`, …

4. **fs_cashflow**

* `company_id`, `fiscal_year`, `fiscal_type`
* `cf_op`, `cf_inv`, `cf_fin`, `capex`, `free_cashflow`, …

5. **ratios**

* `company_id`, `fiscal_year`
* `debt_ratio`, `current_ratio`, `op_margin`, `ebitda_margin`, `interest_coverage`, `sales_growth`, …

6. **meta_source**

* `company_id`, `fiscal_year`, `source`, `file_name`, `download_date`, `note`
